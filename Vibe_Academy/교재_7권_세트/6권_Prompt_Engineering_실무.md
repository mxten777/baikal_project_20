## 1. 프롬프트 엔지니어링 기초 🧠

### 효과적인 프롬프트의 구조

#### CLEAR 원칙
```
C - Context (맥락 제공)
L - Length (적절한 길이)
E - Examples (구체적 예시)
A - Action (명확한 행동 지시)
R - Role (역할 정의)
```

#### 기본 템플릿 구조
```markdown
당신은 [역할]입니다.

[상황/맥락 설명]

다음 요구사항에 따라 [구체적 작업]을 수행해주세요:

요구사항:
1. [요구사항 1]
2. [요구사항 2]
3. [요구사항 3]

출력 형식:
[원하는 출력 형식 명시]

예시:
[구체적인 예시 제공]

제약 조건:
- [제약 조건 1]
- [제약 조건 2]
```

### 프롬프트 품질 평가 기준

#### 평가 체크리스트
```
✅ 역할이 명확하게 정의되어 있는가?
✅ 맥락 정보가 충분히 제공되었는가?
✅ 원하는 출력 형식이 구체적인가?
✅ 예시가 포함되어 있는가?
✅ 제약 조건이 명시되어 있는가?
✅ 모호한 표현이 없는가?
✅ 측정 가능한 기준이 있는가?
```

## 2. 개발용 프롬프트 템플릿 💻

### React 컴포넌트 생성 프롬프트

#### 기본 컴포넌트 템플릿
```markdown
당신은 React 전문 개발자입니다.

다음 요구사항에 따라 React 함수형 컴포넌트를 생성해주세요:

컴포넌트명: [컴포넌트명]
기능: [상세 기능 설명]

요구사항:
1. TypeScript 사용
2. Tailwind CSS 스타일링
3. 반응형 디자인 적용
4. 접근성(a11y) 고려
5. 에러 처리 포함

출력 형식:
- 컴포넌트 파일 코드
- Props 인터페이스 정의
- 사용 예시

제약 조건:
- 함수형 컴포넌트만 사용
- 외부 라이브러리 최소화
- 재사용 가능한 구조
```

#### 실제 사용 예시
```markdown
당신은 React 전문 개발자입니다.

다음 요구사항에 따라 React 함수형 컴포넌트를 생성해주세요:

컴포넌트명: LoginForm
기능: 이메일/비밀번호 로그인 폼

요구사항:
1. TypeScript 사용
2. Tailwind CSS 스타일링
3. 폼 검증 포함 (이메일 형식, 비밀번호 길이)
4. 로딩 상태 표시
5. 에러 메시지 표시
6. 소셜 로그인 버튼 포함 (Google, GitHub)

출력 형식:
- 컴포넌트 파일 코드
- Props 인터페이스 정의
- 사용 예시

제약 조건:
- 함수형 컴포넌트만 사용
- React Hook Form 사용 가능
- 외부 아이콘은 Heroicons 사용
```

## 3. UI/UX 디자인 프롬프트 🎨

### 컴포넌트 디자인 프롬프트

#### UI 컴포넌트 설계 템플릿
```markdown
당신은 UI/UX 디자인 전문가입니다.

다음 요구사항에 따라 [컴포넌트명] 컴포넌트를 디자인해주세요:

컴포넌트: [컴포넌트명]
용도: [사용 목적과 맥락]
타겟 사용자: [사용자 특성]

디자인 요구사항:
1. 브랜드 컬러: [컬러 팔레트]
2. 디자인 스타일: [모던/미니멀/클래식 등]
3. 반응형 지원: [모바일/태블릿/데스크톱]
4. 접근성 준수: [색상 대비, 키보드 탐색 등]
5. 상호작용: [호버, 클릭, 애니메이션]

출력 형식:
- Tailwind CSS 클래스 조합
- 상태별 스타일 정의
- 애니메이션 코드
- 반응형 브레이크포인트

제약 조건:
- Tailwind CSS만 사용
- 커스텀 CSS 최소화
- 성능 최적화 고려
```

#### 실제 사용 예시
```markdown
당신은 UI/UX 디자인 전문가입니다.

다음 요구사항에 따라 가격 카드(Pricing Card) 컴포넌트를 디자인해주세요:

컴포넌트: 가격 카드
용도: SaaS 서비스의 요금제 표시
타겟 사용자: 비즈니스 의사결정권자

디자인 요구사항:
1. 브랜드 컬러: Blue (#3B82F6), Green (#10B981), Gray (#6B7280)
2. 디자인 스타일: 모던 미니멀
3. 반응형 지원: 모바일 1열, 태블릿 2열, 데스크톱 3열
4. 접근성 준수: 색상 대비 4.5:1 이상
5. 상호작용: 호버 시 미세한 상승 효과, 선택 시 테두리 강조

특별 요구사항:
- "인기" 배지가 있는 추천 플랜 강조
- 가격, 기능 목록, CTA 버튼 포함
- 월/연 단위 토글 가능

출력 형식:
- React 컴포넌트 코드
- Tailwind CSS 클래스
- 애니메이션 구현
- 반응형 레이아웃
```

## 4. 비즈니스 기획 프롬프트 📊

### MVP 기획 프롬프트

#### 아이디어 검증 템플릿
```markdown
당신은 린 스타트업 전문가입니다.

다음 비즈니스 아이디어를 분석하고 MVP 기획안을 작성해주세요:

아이디어: [아이디어 설명]
타겟 시장: [시장 정보]

분석 요구사항:
1. 문제-솔루션 핏 검증
2. 타겟 고객 페르소나 정의
3. 경쟁사 분석
4. 차별화 포인트 도출
5. 비즈니스 모델 제안
6. MVP 기능 우선순위
7. 검증 지표 정의

출력 형식:
- 린 캔버스 모델
- 사용자 스토리 맵
- 기능 우선순위 매트릭스
- 검증 실험 계획

제약 조건:
- 2주 내 구현 가능한 MVP
- 최소 예산으로 검증 가능
- 측정 가능한 성공 지표
```

### 사용자 스토리 작성 프롬프트

#### 사용자 스토리 템플릿
```markdown
당신은 제품 기획자입니다.

다음 기능에 대한 사용자 스토리를 작성해주세요:

기능: [기능명]
사용자 유형: [사용자 분류]

요구사항:
1. "As a... I want... So that..." 형식 사용
2. 사용자 관점에서 작성
3. 가치 중심 서술
4. 테스트 가능한 인수 기준 포함
5. 우선순위 라벨링 (Must/Should/Could/Won't)

출력 형식:
- 사용자 스토리 목록
- 인수 기준 (Given-When-Then)
- 우선순위 분류
- 추정 스토리 포인트

제약 조건:
- INVEST 원칙 준수 (Independent, Negotiable, Valuable, Estimable, Small, Testable)
- 기술적 세부사항 배제
- 사용자 가치 중심
```

## 5. 자동화 워크플로우 🤖

### GitHub Actions 자동화

#### CI/CD 파이프라인 프롬프트
```markdown
당신은 DevOps 엔지니어입니다.

다음 프로젝트에 대한 GitHub Actions 워크플로우를 작성해주세요:

프로젝트: [프로젝트 유형]
기술 스택: [기술 스택]
배포 환경: [배포 대상]

워크플로우 요구사항:
1. 코드 품질 검사 (ESLint, Prettier)
2. 타입 체크 (TypeScript)
3. 단위 테스트 실행
4. 빌드 및 배포
5. 환경별 배포 (dev/staging/prod)
6. 알림 시스템 (Slack/Discord)
7. 보안 스캔

출력 형식:
- .github/workflows/*.yml 파일
- 단계별 액션 정의
- 시크릿 변수 목록
- 트리거 조건 설정

제약 조건:
- 비용 효율적 런너 사용
- 병렬 처리 최적화
- 실패 시 롤백 전략
```

## 6. 고급 프롬프트 기법 🎯

### Chain of Thought (CoT) 프롬프트

#### 단계별 사고 유도
```markdown
당신은 [전문가 역할]입니다.

다음 문제를 단계별로 분석하여 해결해주세요:

문제: [복잡한 문제 설명]

해결 과정:
1. 문제 이해 및 정의
2. 관련 요소 파악
3. 가능한 접근 방법 나열
4. 각 방법의 장단점 비교
5. 최적 해결책 선택
6. 구체적 실행 계획

각 단계에서 다음을 포함해주세요:
- 현재 단계의 목적
- 고려한 요소들
- 판단 근거
- 다음 단계로의 연결점

최종 출력:
- 선택한 해결책
- 구현 가이드
- 예상 결과
- 리스크 분석
```

### Few-Shot Learning 프롬프트

#### 예시 기반 학습 유도
```markdown
당신은 [역할]입니다.

다음 예시들을 참고하여 유사한 작업을 수행해주세요:

예시 1:
입력: [예시 입력 1]
출력: [예시 출력 1]

예시 2:
입력: [예시 입력 2]
출력: [예시 출력 2]

예시 3:
입력: [예시 입력 3]
출력: [예시 출력 3]

패턴 분석:
위 예시들에서 공통적으로 나타나는 패턴을 파악하고, 동일한 방식으로 다음 입력을 처리해주세요.

새로운 입력: [실제 처리할 입력]

출력 형식: [원하는 출력 형식]
```

### Role-Based Prompting

#### 전문가 역할 부여
```markdown
## 역할 정의
당신은 [구체적 전문 분야]에서 [경험 년수]년 경험을 가진 [직책]입니다.

## 전문성 요소
- 주요 기술: [기술 목록]
- 프로젝트 경험: [프로젝트 유형]
- 업계 이해도: [도메인 지식]

## 작업 스타일
```

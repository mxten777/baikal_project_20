## 1. MVP 기획의 핵심 원리 🎯

### MVP 기획 3원칙

#### 원칙 1: 문제 중심 사고
```
❌ 잘못된 접근: "멋진 기능을 만들어보자"
✅ 올바른 접근: "이 문제를 어떻게 해결할까?"
```

#### 원칙 2: 최소 범위 정의
```
❌ 처음부터 완벽한 서비스
✅ 핵심 가치 1개만 제공
```

#### 원칙 3: 검증 가능한 가설
```
❌ "사용자가 좋아할 것이다"
✅ "30일 내 100명이 사용할 것이다"
```

### MVP 성공 요소 분석
| 요소 | 중요도 | 설명 |
|------|--------|------|
| **문제 명확성** | ⭐⭐⭐⭐⭐ | 해결하려는 문제가 명확한가? |
| **타겟 사용자** | ⭐⭐⭐⭐⭐ | 누가 사용할지 구체적인가? |
| **핵심 기능** | ⭐⭐⭐⭐ | 1~3개 기능으로 제한했는가? |
| **기술 난이도** | ⭐⭐⭐ | 2주 내 구현 가능한가? |
| **확장성** | ⭐⭐ | 향후 발전 가능성이 있는가? |

## 2. 1페이지 프롬프트 작성법 📝

### 프롬프트 구조 템플릿

```markdown
# [프로젝트명] MVP 프롬프트

## 🎯 문제 정의 (2줄)
- 누가: [타겟 사용자]
- 무엇을: [어떤 문제로 불편해하는지]

## ✨ 해결책 (3줄)
- 핵심 기능 1: [가장 중요한 기능]
- 핵심 기능 2: [두 번째 중요한 기능]
- 핵심 기능 3: [세 번째 중요한 기능]

## 🎨 UI/UX 방향 (2줄)
- 디자인 스타일: [간단/모던/클래식 등]
- 사용자 플로우: [3단계 이내]

## 🔧 기술 스택
- Frontend: Vite + React + Tailwind
- Backend: Firebase (Auth + Firestore)
- Deploy: Vercel

## 📊 성공 지표 (1줄)
- [구체적이고 측정 가능한 목표]
```

### 실제 작성 예시: 로또 번호 생성기

```markdown
# 🎲 Smart Lotto MVP 프롬프트

## 🎯 문제 정의
- 누가: 로또 구매자들이
- 무엇을: 매번 같은 번호를 선택하거나 번호 선택에 고민하는 문제

## ✨ 해결책
- 핵심 기능 1: AI 기반 번호 추천 (통계 + 랜덤)
- 핵심 기능 2: 개인 행운 번호 저장 및 관리
- 핵심 기능 3: 당첨 확률 시뮬레이션

## 🎨 UI/UX 방향
- 디자인 스타일: 간단하고 직관적인 카드 UI
- 사용자 플로우: 클릭 → 번호생성 → 저장/공유

## 🔧 기술 스택
- Frontend: Vite + React + Tailwind
- Backend: Firebase (Firestore)
- Deploy: Vercel

## 📊 성공 지표
- 30일 내 일일 사용자 50명 달성
```

## 3. AI 프롬프트 엔지니어링 🤖

### 효과적인 AI 프롬프트 작성법

#### 구조화된 프롬프트
```
당신은 [역할]입니다.
[구체적인 요구사항]을 만들어주세요.

조건:
1. [조건 1]
2. [조건 2]
3. [조건 3]

출력 형식:
- [원하는 형식 설명]

예시:
[구체적인 예시 제공]
```

#### MVP 개발용 프롬프트 템플릿

**UI 컴포넌트 생성 프롬프트:**
```
당신은 React 개발자입니다.
[기능 설명]을 위한 React 컴포넌트를 만들어주세요.

조건:
1. Tailwind CSS 사용
2. 반응형 디자인
3. 접근성 고려
4. TypeScript 사용 (선택)

파일 구조:
- 컴포넌트 파일
- CSS 클래스명 명명규칙
- Props 인터페이스 정의
```

**Firebase 연동 프롬프트:**
```
Firebase Firestore를 사용해서 [데이터 구조] CRUD 기능을 구현해주세요.

요구사항:
1. 컬렉션명: [collection_name]
2. 문서 구조: [document_structure]
3. 실시간 업데이트 적용
4. 에러 처리 포함

예상 사용 시나리오:
[구체적인 사용 사례 설명]
```

## 4. 실전 MVP 기획 템플릿 📋

### 4-Step MVP 기획 프로세스

#### Step 1: 브레인스토밍 (10분)
```
1. 오늘 불편했던 일 3가지 적기
2. 내가 자주 사용하는 앱의 아쉬운 점
3. 주변 사람들이 자주 하는 불평
```

#### Step 2: 문제 선택 및 구체화 (10분)
```
선택 기준:
- 나도 겪는 문제인가?
- 간단한 해결책이 있는가?
- 2주 안에 만들 수 있는가?
```

#### Step 3: 솔루션 설계 (15분)
```
핵심 질문:
- 최소한의 기능으로 문제를 해결할 수 있는가?
- 사용자가 5분 안에 이해할 수 있는가?
- 첫 사용에서 가치를 느낄 수 있는가?
```

#### Step 4: 검증 계획 수립 (5분)
```
검증 방법:
- 어떻게 사용자 피드백을 받을 것인가?
- 성공/실패를 어떻게 판단할 것인가?
- 다음 단계 계획은?
```

## 5. 40개 MVP 사례 분석 📊

### 카테고리별 성공 사례

#### 🎯 생산성 도구 (성공률: 85%)
1. **Pomodoro Timer Pro**
   - 문제: 집중력 관리의 어려움
   - 해결: 25분 집중 + 5분 휴식 타이머
   - 핵심 기능: 타이머, 통계, 알림
   - 성과: 월 500+ 사용자

2. **Daily Habit Tracker**
   - 문제: 습관 형성의 어려움
   - 해결: 시각적 진행률 추적
   - 핵심 기능: 체크리스트, 연속 기록, 리워드
   - 성과: 월 300+ 사용자

#### 💰 금융/투자 도구 (성공률: 70%)
3. **Crypto Price Alert**
   - 문제: 암호화폐 가격 모니터링
   - 해결: 실시간 알림 시스템
   - 핵심 기능: 가격 추적, 알림 설정, 포트폴리오
   - 성과: 월 800+ 사용자

4. **Stock Dividend Tracker**
   - 문제: 배당 수익 관리의 복잡함
   - 해결: 자동 배당 일정 추적
   - 핵심 기능: 포트폴리오, 배당 캘린더, 세금 계산
   - 성과: 월 200+ 사용자

#### 🎲 엔터테인먼트 (성공률: 60%)
5. **Random Restaurant Picker**
   - 문제: 식당 선택의 어려움
   - 해결: 위치 기반 랜덤 추천
   - 핵심 기능: 위치 인식, 필터링, 리뷰 연동
   - 성과: 월 1000+ 사용자

### 실패 사례 분석

#### 실패 원인 Top 3
1. **문제가 명확하지 않음** (40%)
   - 예: "모든 걸 다 할 수 있는 앱"
   - 해결: 하나의 명확한 문제에 집중

2. **과도한 기능 욕심** (35%)
   - 예: 첫 버전에 10개 기능 구현
   - 해결: 핵심 기능 1~3개로 제한

3. **타겟 사용자 불명확** (25%)
   - 예: "모든 사람이 사용할 앱"
   - 해결: 구체적인 사용자 페르소나 정의

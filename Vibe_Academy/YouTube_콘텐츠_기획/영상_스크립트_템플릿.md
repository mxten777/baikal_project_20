## 📝 스크립트 작성 원칙

### 구조 템플릿 (HOOK-BODY-CTA)

```markdown
## [영상 제목]
**길이**: [예상 시간]분
**시리즈**: [시리즈명]
**에피소드**: EP[번호]

---

### 🎯 HOOK (첫 30초)
**목적**: 시청자 관심 끌기, 이탈 방지

#### 오프닝 (0-15초)
- 임팩트 있는 문제 제기 또는 결과 미리보기
- "만약 당신이 X를 Y로 할 수 있다면?"
- "오늘 X분 만에 Y를 완성해보겠습니다"

#### 가치 제안 (15-30초)  
- 이 영상을 보면 얻을 수 있는 것
- 구체적인 혜택이나 결과물
- "이 영상 끝까지 보시면..."

### 📚 BODY (메인 콘텐츠)
**목적**: 핵심 가치 전달, 교육/정보 제공

#### 섹션 1: 배경/문제 설명 (전체의 20%)
#### 섹션 2: 솔루션 설명 (전체의 60%)  
#### 섹션 3: 결과/검증 (전체의 20%)

### 🚀 CTA (마지막 1분)
**목적**: 행동 유도, 채널 성장

#### 요약 (30초)
- 핵심 내용 3줄 요약
- 주요 인사이트 강조

#### 행동 유도 (30초)
- 구독/좋아요/알림 요청
- 댓글 질문/과제 제시
- 다음 영상 예고
```

## 🎭 시리즈별 스크립트 템플릿

### 📺 "8주 완성! Vibe Coding 실전 시리즈" 템플릿

```markdown
## [주차]-[회차]: [구체적 주제]
**길이**: 15-25분
**대상**: Vibe Academy 수강생 + 일반 시청자

---

### 🎯 HOOK (0-1분)

#### 오프닝 (0-30초)
안녕하세요, 바이칼 시스템즈 [이름]입니다.
[이번 주제와 관련된 임팩트 있는 질문이나 상황]

예: "React를 처음 배우는데 어디서부터 시작해야 할지 
막막하다면, 오늘 영상이 완벽한 답이 될 것입니다."

#### 가치 제안 (30-60초)
오늘 이 영상을 끝까지 보시면:
- [구체적 학습 결과 1]
- [구체적 학습 결과 2]  
- [구체적 학습 결과 3]

실제로 지난 6개월간 40개 MVP를 만들면서 
가장 효과적이었던 방법들만 엄선해서 
단계별로 알려드리겠습니다.

### 📚 BODY (1-24분)

#### 섹션 1: 이론 설명 (3-5분)
**화면**: 슬라이드 + 얼굴
- 개념 정의
- 왜 중요한지 설명
- 전체 맥락에서의 위치

#### 섹션 2: 실습 과정 (15-18분)
**화면**: 화면 공유 + 작은 얼굴 창
- 단계별 실습 진행
- 코드 작성 과정 실시간 시연
- 자주 발생하는 오류와 해결법
- 중간중간 핵심 포인트 강조

#### 섹션 3: 결과 확인 및 활용 (1-2분)
**화면**: 완성된 결과물
- 완성된 기능 시연
- 추가 개선 가능성
- 다음 단계 연결점

### 🚀 CTA (24-25분)

#### 요약 (30초)
오늘 우리가 함께 한 것은:
1. [핵심 내용 1]
2. [핵심 내용 2] 
3. [핵심 내용 3]

#### 행동 유도 (30초)
구독과 좋아요는 더 좋은 콘텐츠 제작에 큰 도움이 됩니다.
댓글로는 오늘 실습하면서 어려웠던 부분이나 
추가로 궁금한 점을 남겨주세요.

다음 시간에는 [다음 주제 예고]에 대해 알아보겠습니다.
그럼 다음 영상에서 만나요!

---

### 📝 연출 및 편집 노트

#### 중요한 편집 포인트
- 1:30 - 첫 번째 "왜 중요한지" 자막 강조
- 5:00 - 실습 시작 알림 그래픽
- 12:00 - 중간 체크포인트 (진행률 표시)
- 20:00 - 완성 임박 알림
- 23:00 - 최종 결과 하이라이트

#### B-Roll 삽입 포인트
- 개념 설명 시: 관련 이미지, 다이어그램
- 코딩 과정: 타이핑하는 손, 전체 모니터 화면
- 결과 확인: 다양한 각도에서 완성품 촬영

#### 자막 가이드라인
- 핵심 키워드: 노란색 하이라이트
- 중요한 팁: 빨간색 박스
- 코드나 명령어: 고정폭 폰트, 회색 배경
- 주의사항: 느낌표와 함께 주황색
```

### 💡 "Prompt Clinic" 템플릿

```markdown
## Prompt Clinic EP[번호]: [구체적 해결 문제]
**길이**: 5-8분
**포맷**: 빠른 해결책 제시

---

### 🎯 HOOK (0-30초)

#### 문제 상황 제시 (0-15초)
"[구체적인 개발 상황] 때문에 고민이신가요?"
또는
"[일반적인 개발 고민]으로 시간을 낭비하고 계신가요?"

#### 솔루션 예고 (15-30초)
"오늘 Prompt Clinic에서는 이 문제를 
단 5분 만에 해결하는 마법의 프롬프트를 알려드립니다."

### 📚 BODY (30초-7분)

#### 프롬프트 소개 (30초-2분)
**화면**: 프롬프트 텍스트 전체 화면 표시
- 프롬프트 전문 읽어주기
- 각 부분이 왜 중요한지 설명
- 핵심 키워드 강조

#### 실행 과정 (2-5분)
**화면**: AI 도구 화면 공유
- 프롬프트 입력
- AI 응답 확인
- VS Code에 적용
- 실행 결과 확인

#### 결과 분석 및 팁 (5-7분)
**화면**: 완성된 결과물
- 생성된 코드 품질 평가
- 추가 개선 방법
- 비슷한 상황에서 응용하는 법

### 🚀 CTA (7-8분)

#### 요약 (30초)
"오늘의 마법 프롬프트로 [문제]를 해결했습니다!"

#### 행동 유도 (30초)
"여러분도 이 프롬프트를 사용해보시고,
결과를 댓글로 공유해주세요!
다음 Prompt Clinic에서 다룰 주제도 댓글로 신청받습니다."

---

### 📝 제작 노트

#### 편집 스타일
- **빠른 템포**: 불필요한 정적 구간 제거
- **시각적 강조**: 프롬프트 부분은 큰 글씨로
- **결과 하이라이트**: Before/After 비교
- **타이머**: 실제로 걸린 시간 표시

#### 썸네일 요소
- 문제 상황 이미지
- "5분 해결" 텍스트
- 프롬프트 일부 미리보기
- 놀란 표정의 얼굴
```

### 🧱 "Behind the MVP" 템플릿

```markdown
## Behind the MVP EP[번호]: [MVP 이름] 제작기
**길이**: 15-20분
**스타일**: 다큐멘터리 + 회고

---

### 🎯 HOOK (0-2분)

#### 완성품 시연 (0-90초)
**화면**: 완성된 MVP 화면
- 실제 동작하는 모습
- 핵심 기능 간단 시연
- 성과 지표 공개 (사용자 수, 반응 등)

#### 제작 기간과 도전 (90-120초)
"이 서비스를 만드는 데 걸린 시간은 단 [X]일.
하지만 그 과정에는 예상치 못한 도전들이 있었습니다."

### 📚 BODY (2-18분)

#### Phase 1: 아이디어와 기획 (2-6분)
**화면**: 기획 문서, 스케치, 경쟁사 분석
- 문제를 발견한 계기
- 아이디어 구체화 과정
- 1페이지 프롬프트 작성
- 기술 스택 선택 이유

#### Phase 2: 개발 과정 (6-14분)
**화면**: 실제 개발 과정 녹화 영상
- Day by day 개발 일지
- 주요 기능 구현 과정
- 마주친 기술적 문제들
- AI 도구 활용 사례
- 예상보다 어려웠던/쉬웠던 부분

#### Phase 3: 배포와 피드백 (14-17분)
**화면**: 배포 과정, 사용자 반응
- 배포 과정에서의 문제
- 첫 사용자 반응
- 예상과 다른 사용 패턴
- 긴급 수정 사항들

#### Phase 4: 성과와 교훈 (17-18분)
**화면**: 성과 지표, 회고 노트
- 정량적 성과 (사용자, 매출 등)
- 정성적 성과 (피드백, 만족도)
- 다음에 개선하고 싶은 점
- 다른 개발자들에게 주는 팁

### 🚀 CTA (18-20분)

#### 핵심 인사이트 (60초)
"이 프로젝트를 통해 배운 가장 중요한 교훈 3가지"

#### 행동 유도 (60초)
"여러분도 만들어보고 싶은 MVP가 있다면 
댓글로 공유해주세요. 
다음 Behind the MVP 주인공이 될 수도 있습니다!"

---

### 📝 제작 가이드

#### 사전 준비물
- [ ] 개발 과정 전체 스크린 녹화
- [ ] 각 단계별 스크린샷
- [ ] 완성품 데모 영상
- [ ] 성과 지표 자료
- [ ] 사용자 피드백 캡처

#### 인터뷰 질문 리스트
1. 이 아이디어는 어떻게 떠올리게 되었나요?
2. 개발 과정에서 가장 어려웠던 점은?
3. AI 도구가 어떻게 도움이 되었나요?
4. 예상과 다른 사용자 반응이 있었나요?
5. 다음에 비슷한 프로젝트를 한다면 무엇을 다르게 할 건가요?
```
